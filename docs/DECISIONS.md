# Decisions

Architectural decisions for possession. Lock these in until explicitly
revisited.

| ID | Decision                                                      | Rationale |
|----|---------------------------------------------------------------|-----------|
| D1 | Single static binary, Go 1.26                                  | Easy distribution; one runtime; native concurrency for the future replay engine. |
| D2 | Subprocess-invokable: stable `--json` contract on `parse` (and later `scan`) | Pho3nix and other harnesses shell out rather than embed; keeps possession's license isolation clean. |
| D3 | HAR + curl as input formats                                   | HAR comes free from every browser devtools; curl is the lingua franca of API capture. Postman / OpenAPI deferred. |
| D4 | Path templating uses simple table-driven heuristics, not LLMs  | Deterministic, fast, debuggable, no external dependency. |
| D5 | Role matrix is YAML, validated up front, errors aggregated     | Users edit it directly; one run surfaces every problem. |
| D6 | Glob dialect is a tiny custom doublestar (see ARCHITECTURE.md) | Avoids `gobwas/glob` dep; covers `*`, `**`, `?` which is all we need for path scoping. |
| D7 | **License: AGPL-3.0-only** (override of the original packet)   | The original brief proposed MIT. The human explicitly overrode this and chose AGPL-3.0 for maximum copyleft protection. The AGPL network clause matters because possession may be reused inside SaaS products (e.g. Pho3nix). Per D2, downstream tools invoke possession as a subprocess, so they do NOT pick up AGPL obligations on their own source — only modifications to possession itself must be shared. This deliberately keeps Pho3nix's own code unaffected while protecting the tool. |
| D8  | v1.0 mutators: strip-auth, swap-identity, downgrade-role, drop-cookie, strip-token | Cover authn-bypass, IDOR / cross-user, privilege escalation, auth-dependency mapping, and token-vs-cookie enforcement. JWT mutators deferred to P4. |
| D9  | Replay-as-self is the baseline anchor                                   | The matrix is N identities (self included), not N−1 pairs. Replaying an identity as itself gives the comparison anchor for P3's detection evaluator. |
| D10 | Refresh-hook failure aborts that identity only                          | Variants for the failed identity are marked Inconclusive; loud warning logged; the run continues with other identities. A single credential outage must not invalidate the whole scan. |
| D11 | Variant cap 10000 default with deterministic generation order           | Bounded blast radius for accidentally-huge captures. Order: endpoints (method, pathTemplate) → representative sample (smallest ID) → mutator (declaration order) → identity (rank, name). Variant ID = sha256(epKey + mutator + identity + canonical_detail)[:16]. |
| D12 | Body size cap 5MB default                                               | P3 needs bodies in memory for similarity scoring; 5MB keeps that affordable. Larger bodies are truncated and Response.Truncated is set. |
| D13 | Refresh extraction selector is minimal dotted-path                      | Supports `$.a.b.c` and `$.a[0].b` only. No wildcards, filters, or recursive descent. Small surface area means small bug surface; full JSONPath is overkill for refresh hooks. |
| D14 | `scan` is self-contained                                                | Runs parse → normalize → scope filter → variant gen → replay in one process. Avoids re-piping `parse --json` into a separate replay tool. |
| D15 | Per-host token-bucket limiter + bounded concurrency + adaptive backoff  | Default 10 req/s/host, concurrency 5. 429/503 honors Retry-After then falls back to exponential 1s/2s/4s, up to 3 retries then errored. `--no-limit` requires a loud warning. Implemented with `golang.org/x/time/rate`. |
| D16 | Pluggable Evaluator interface; ship ComparativeEvaluator only for v1.0   | Realizes D4. One implementation now (Autorize-style: replay-as-self baseline vs replay-as-other comparison) with a clean interface so a future AssertionEvaluator (AuthMatrix-style declarative expectations) can drop in without rework. |
| D17 | Capture-owner attribution by credential match                            | For each CapturedRequest, match its auth components against the matrix identities (bearer/cookie/header/basic-username). First match wins; ties broken by (rank asc, name asc) with a recorded warning. No match falls back to highest-rank identity with attribution `fallback-highest-rank`. Reuses mutate's auth-component heuristic; no reimplementation. |
| D18 | Calibrated baseline — owner self-replay fired N times                    | Default `--baseline-samples 3`, clamped 1..10. Mean pairwise similarity sets per-endpoint stability. Endpoints with stability < `NoisyEndpointThreshold` (0.70) are flagged `noisy` and all verdicts capped at `suspected`. Per-endpoint `effThreshold = stability - SimilarityMargin`, clamped to `[MinThreshold, 1.0]`. N=1 skips calibration with `DefaultThreshold=0.90` and emits an endpoint note. Baseline-not-2xx ⇒ all variants on that endpoint are `inconclusive` with a loud per-endpoint warning. |
| D19 | Four verdicts: bypass, suspected, enforced, inconclusive                 | `bypass` = high-confidence finding. `suspected` = medium, needs review. `enforced` = authz is working, no finding. `inconclusive` = couldn't judge (refresh failure, transport error, baseline failure). Findings emitted only for `bypass` and `suspected`. |
| D20 | Optional per-identity `markers` in role matrix                           | Additive YAML schema: each identity may declare a `markers` list of strings that uniquely identify that identity's data (email, account id, display name). When present, `reflectedOwner` (variant body contains the *resource owner's* marker) is the strongest bypass signal; `reflectedActor` (variant body contains *only the acting identity's own* marker) is a strong benign signal. No markers ⇒ those signals are inert. |
| D21 | drop-cookie one-at-a-time; strip-token bearer vs csrf stays separate     | No combined `drop-all-cookies` or `drop-bearer-and-csrf` variants. Each variant isolates exactly one auth component so any resulting bypass attributes cleanly to that component. Combined cases inflate variant counts without adding diagnostic value. |
| D22 | ASVS v5.0.0 chapter V8 (Authorization) for control mapping               | NOT v4.0.x. IDs emitted in `v5.0.0-8.2.2` form. Fixed mapping by Finding.Class: authn-bypass⇒8.3.1, idor⇒8.2.2, idor-cross-tenant⇒8.4.1+8.2.2, privesc⇒8.2.1, auth-dependency⇒8.3.1. Suspected verdicts drop severity one notch (critical→high, high→medium, low→info). |
| D23 | All tuning constants live in internal/detect/tuning.go                   | Every threshold, weight, regex, volatile-key list, and ASVS map is declared in one file with comments marking them as calibration starting points. Zero magic numbers anywhere else under internal/detect/. Makes calibration a one-file diff. |
| D24 | v1.0 JWT attack set: alg-none, sig-strip, claim-tamper, resign-weak-key  | Covers the four practical JWT bypass shapes (unsigned token, missing-sig, claim tampering on a token the verifier doesn't check, weak-secret HMAC). RS256→HS256 alg-confusion, kid injection, JWKS poisoning, full HMAC cracking → v1.1. |
| D25 | JWT lib: golang-jwt/jwt/v5 declared; Decode hand-rolled for leniency      | The dependency is declared (per the brief) but Decode is a custom parser because golang-jwt's lenient-parse modes changed across v5 minor versions and we need stable behavior on adversarial inputs (alg=none, empty sigs). encode.go is hand-rolled by design — assembling malformed tokens is the whole point of the mutators. Documented in internal/jwt/encode.go header comment. |
| D26 | Reporter interface + sarif/json/human formatters; --report selects        | The Reporter interface is one method (Render). `human` is the default because the most common consumer is a human reading the terminal output. JSON is the Pho3nix-facing contract (D2): deterministic, byte-stable, indented. SARIF is the GitHub-Code-Scanning contract: SARIF 2.1.0 via owenrumney/go-sarif/v3 with rule helpUri to ASVS and partialFingerprints for dedupe. |
| D27 | SARIF lib: owenrumney/go-sarif/v3 at SARIF 2.1.0                          | Round-trippable through the same library. One rule per finding class with ASVS in helpUri + property bag; one result per finding with severity→level mapping (critical/high→error, medium→warning, low/info→note) and Finding.ID as partialFingerprints key. Gate F: V9 IDs intentionally omitted — see D28's sibling note in the §10 report. |
| D28 | Cross-rank swap-identity ⇒ suspected with cross-rank-swap note            | Override of Packet 3's `enforced` short-circuit. The verdict ladder runs as normal, then a post-ladder cap downgrades any bypass to suspected with a typed `cross-rank-swap` note (confidence is multiplied by AmbiguousPenalty). Rationale: a higher-rank actor reading the owner's data MAY be a designed-in admin override or MAY be the bypass the user is hunting for — we can't know, so we report it at reduced confidence rather than silently dropping it. Filter A (same-identity replay short-circuit) is preserved unchanged. |
| D29 | `endpoint_notes` are typed (detect.EndpointNote enum), not prefix strings | NoteCode enum: calibration-skipped, noisy-endpoint, baseline-not-2xx, refresh-failed, min-confidence, cross-rank-swap. Each note has a code, payload, and pre-rendered Message. Reporters render the Message; tooling can switch on Code. Replaces P3's prefix-tagged free strings. |
| D30 | Mutation.Class set ONCE at variant generation                             | Each mutator (P2 + the four P4 JWT mutators) sets Mutation.Class at the moment it constructs the Variant. detect/finding.go reads Variant.Mutation.Class; MutatorClass() is the fallback for variants built without a mutator (baseline-self, tests). scan.go no longer re-derives. |
| D31 | `idor-cross-tenant` stays dormant in v1.0                                 | The code path is wired (ASVS map + class branch) but no tenant marker support in the role-matrix schema yet, so it can never fire. v1.1 will add a per-identity tenant field and the class will activate automatically. |
| D32 | JWT helper Decode is a custom parser, not golang-jwt's                    | golang-jwt/jwt/v5 is in go.mod for its types/conventions, but our Decode is a 30-line custom parser explicitly designed to succeed on alg=none and stripped-sig tokens without requiring opt-in parser flags that change between minor versions. The brief's intent ("be lenient") is preserved, the surface is smaller, and the unit tests assert the exact behavior. |
| D33 | P5 deep JWT mutators use ephemeral key generation per Generate() call     | jwt-jwks-spoof generates a fresh RSA-2048 key pair on every call to Generate(). This keeps variants independent and avoids shared key-material state between corpus runs. Performance cost is ~10ms per Generate() call — acceptable for a security scanner. Determinism is intentional: each scan produces fresh keys, which is correct behavior for an attacker-simulation tool. |
| D34 | alg-confusion uses DER bytes of PEM block as HMAC secret                  | The classic CVE pattern: re-encode the public key as raw DER bytes and use those as the HMAC secret. We use block.Bytes directly (no additional marshaling) so the bytes match exactly what a real attacker would use (openssl asn1parse output). Tested against a vulnapp that accepts unsigned tokens. |
| D35 | jwt-hmac-crack: one variant per location, break on first cracked secret    | The mutator emits at most one variant per token location (the first cracked secret). Multiple cracked-secret variants for the same location add no diagnostic value — the finding is "secret is weak", not a per-secret report. The Detail field records the cracked secret. HmacCrackMaxAttempts caps total attempts at 500 per location. |
| D36 | target.jwt schema is additive and optional                                 | Adding target.jwt to the matrix is optional. Absent → alg-confusion is skipped with no error (the mutator returns nil silently). This preserves backward compatibility for all v1.0 matrices. The skip is logged in the variant plan (dry-run output shows skipped mutators). |
| D37 | AssertionEvaluator reports bypass with high confidence (0.97)              | Assertion-derived bypass has higher confidence than comparative-derived bypass (max ~0.95) because the expectation is explicit — no inference needed. This also makes assertion bypasses easier to filter/prioritize in downstream tooling. |
| D38 | Glob precedence: longest pattern string wins                               | When multiple assertion patterns match an endpoint, the one with the longest string representation wins. This is a simple, predictable rule that handles the common "general catch-all + specific exception" case without requiring a precedence score. Ties broken by declaration order (stable). |
| D39 | broken-deny surfaces as suspected (not bypass)                             | access-denied-but-allow-expected is a functionality bug (overly-strict), not a security vulnerability. Surfacing it as suspected keeps it in the report (visible) without alarming security reviewers who are scanning for bypass findings. |
| D40 | --evaluator comparative is the default                                     | Preserves v1.0 behavior — users who haven't written an assertions block won't be surprised by an error. Assertion mode requires explicit opt-in. |
| D41 | Flows are Tier-2 multi-step refresh hooks with {name} interpolation        | A flow is an ordered list of steps; each step issues one request and extracts values available to later steps via {name} interpolation. Identities reference a flow by name; the replay engine executes the flow once per identity (caching) and re-runs volatile steps per batch for CSRF/nonce freshness. Same D10 failure policy as Tier-1: flow failure → that identity's variants inconclusive. |
| D42 | Interpolation regex restricted to valid identifiers                        | {name} placeholders match [A-Za-z][A-Za-z0-9_-]* only, preventing false matches inside JSON bodies like {"key":"value"} where { and } are JSON syntax. This is a pragmatic restriction; full template engines are out of scope. |
| D43 | Flow extracted values are injected via explicit Inject directives          | Each FlowExtraction has an optional Inject field (same shape as Tier-1 Extraction.Inject) specifying where to place the value in variant requests. Without an Inject directive the value is only available for {name} interpolation within the flow itself (not automatically injected into variants). |
| D44 | Volatile step re-run uses ExecuteFrom starting at the volatile head index  | When a flow has volatile extractions (nonce, CSRF token), ExecuteFrom re-runs the flow from the first volatile step before each variant. This ensures fresh anti-replay values while avoiding full flow re-execution. The volatile head index is determined by the first step containing a volatile: true extraction. |
| D45 | OAuth2 token acquisition is a flow step type, not a separate system        | Adding an oauth2: field to a FlowStep issues the token request (client_credentials or refresh_token grants) and extracts the access_token into the flow variable bag. The access_token can then be injected into variant requests via the Inject directive. SAML, PKCE, device_code, and other OAuth flows are deferred to v1.2. |
| D46 | Cross-tenant detection activates via per-identity Tenant field              | Activates D31's dormant idor-cross-tenant path. swap-identity variants compare the captured request's owner tenant (resolved by bearer token match) against the actor's Tenant field. Cross-tenant → class=idor-cross-tenant, critical severity, ASVS v5.0.0-8.4.1+8.2.2. Identities without Tenant tag are not considered for cross-tenant detection. |

## Gate-D additive shape changes for Packet 3

Three additive changes to Packet-1/Packet-2 public types that Packet 3 introduces:

1. **model.Endpoint** gains `OwnerIdentity *Identity` and `OwnerAttribution string` fields (D17 attribution). Existing zero values remain valid.
2. **model.Response** gains `BodySHA256 string` field, always populated by the replay engine for any non-empty body. Existing consumers ignore the new field.
3. **model.Identity** gains `Markers []string` (D20). Optional; existing matrices that omit it parse identically.

Plus model.Variant.Mutation.Class is now populated (by the evaluator, not by Packet-2 mutators — keeps the mutator code frozen).

Plus model.Finding and model.Evidence are filled out per §5 of the Packet 3 brief.

## D7 override note (audit trail)

The original Packet 1 brief, under section D7, specified MIT. Before
implementation began, the project owner explicitly overrode this in
favor of AGPL-3.0-only. The override is captured here, in the LICENSE
file, in the README, and in the Packet 1 final report so that the
deviation is visible at every entry point.
